# ADR-005: v0.5.0 F9 — Mock dogfood + marketplace verification (라이브 install 다운그레이드)

- **Status**: Accepted
- **Date**: 2026-04-26
- **PR**: feat/v0.5.0-pm-growth-tracks (Ship 시 채움)
- **Linked SPEC**: `docs/specs/new-tracks-pm-growth.md` (Accepted 2026-04-25)
- **Classification**: Major CR — SPEC §3.1 F9 (4/4 Pass) + §5.2 (E2E Mock 금지) 영향.

---

## Context

`docs/specs/new-tracks-pm-growth.md` §3.1 F9 + §5.2는 신규 Track 검증 절차로 **`growth-marketing` Track 1회 라이브 install**을 명시:

> | F9 | dogfood 실측 1회 — 신규 2 Track 중 1개 라이브 install (`growth-marketing` 권장 — 자산 수 5종으로 매트릭스 부담 큰 쪽) | E2E Mock 금지 (test-policy.md Dev-Prod Parity 정신) | 4/4 |

Build/Verify/Review 단계에서 reviewer subagent (5축 리뷰)가 다음 사항을 MEDIUM 이슈로 지적:

- 본 세션 dogfood (`docs/dogfood/cli-dogfood-2026-04-26-v0.5.0.md`)는 라이브 install 미수행. marketplace verification + Mock dogfood (매트릭스 437 tests PASS)로 대체.
- SPEC F9 미충족 → Major CR 또는 라이브 실행 둘 중 하나 필요.

라이브 install이 다음 위험을 가짐을 사용자/agent가 평가:

1. **D16 보호 영역 영향 가능성** — `claude plugin marketplace add` + `claude plugin install`은 글로벌 plugin 캐시(`~/.claude/plugins/`, marketplace registry)를 수정. tmp dir 격리는 프로젝트 스코프 자산만 격리, 글로벌 plugin 캐시는 격리 불가.
2. **v0.5.0 작업 컨텍스트 외 시스템 상태 변경** — 본 SPEC §3.3 DO NOT CHANGE에 `~/.claude/` 명시.

## Decision

**F9를 다음으로 다운그레이드**:

- **F9-revised**: `growth-marketing` Track 검증 = (a) marketplace verification (`gh api` 8 plugin id 등록 + marketplace name `claude-code-skills` 일치) + (b) Mock dogfood (vitest 매트릭스 + condition 평가 437 tests PASS).
- **라이브 install은 v0.6+ 또는 사용자 명시 승인 후 별도 세션**으로 이월. SPEC §3.4 OQ4와 통합.

§5.2 "Mock 금지" 항목은 다음으로 수정:

- `growth-marketing` 라이브 install — **이월 (ADR-005)**. v0.6+ 별도 세션 또는 사용자 승인 시 즉시 실행.

## Alternatives

### Alt-1: tmp dir 라이브 install 즉시 실행

- **장점**: SPEC F9 원안 충족.
- **단점**: 글로벌 plugin 캐시 수정 — D16 보호 영역 영향 가능. 본 SPEC §3.3 DO NOT CHANGE 위반 risk.
- **기각 사유**: D16 우선. 본 SPEC의 DO NOT CHANGE 보호 영역과 직접 충돌.

### Alt-2: F9 삭제 (검증 절차 자체 제거)

- **단점**: 검증 누락 → install 시점 실패 위험 증가.
- **기각 사유**: marketplace verification + Mock dogfood로 install 실패 risk를 낮춤. 검증 자체는 유지.

### Alt-3: 본 ADR 채택 (Mock + marketplace verification 충분 결정)

- **장점**: D16 보호 + 검증 누락 0 + lean.
- **검증 근거**:
  - marketplace name `claude-code-skills` = 우리 pluginId 형식 `<name>@claude-code-skills`와 정확히 일치.
  - 신규 7 plugin id (pm-skills, product-skills, marketing-skills, content-creator, demand-gen, research-summarizer, karpathy-coder) 모두 marketplace에 등록 확인.
  - vitest 매트릭스 — 11×5=55 + invariant 6 = 61 PASS, 신규 9 case (project-management/growth-marketing 매핑 + business-growth 합집합 + karpathy-coder 제외/포함).
  - condition 평가 SSOT 통일 (`shouldInstallAsset` `has-dev-track` → `track-match.hasDevTrack`) + 신규 unit test 4 case.
- **잔여 risk**: install 시점에 marketplace 응답 변동 (네트워크 / upstream 변경). 그러나 이 risk는 라이브 install 1회 실측으로도 영구히 보장되지 않음 (시점성).
- **선택**.

## Consequences

### 즉시
- SPEC `docs/specs/new-tracks-pm-growth.md` §3.1 F9 + §3.4 OQ4 + §5.2 본문에 본 ADR-005 reference 추가 (별도 commit).
- todo `docs/plans/new-tracks-pm-growth-todo.md` P4-T5 항목 — "라이브 dogfood = ADR-005에 따라 mock + marketplace verification으로 충족, 라이브는 v0.6+ 이월" 명시.
- v0.5.0 Ship 게이트 통과 (CRITICAL 0).

### 후속
- v0.6+ 첫 SPEC에 OQ4 후속 항목 등재 — 라이브 dogfood 절차 (tmp dir + 글로벌 cache snapshot 전후 비교 또는 사용자 명시 승인).
- 사용자가 본 세션 후 임의로 라이브 install 시도 → 절차는 dogfood 리포트 §5에 명시됨.

### Risk
- marketplace upstream 변경 (plugin id 변경 / removal) 시 우리 매트릭스 테스트가 검출 못함. 그러나 install 실행 시 `external-installer.ts` warn-skip 모드로 fall-through하므로 사용자 경험은 partial install + 누락 보고. abort 아님.

### Reversibility
- 본 ADR 결정은 본 세션 v0.5.0 한정. v0.6+에서 라이브 dogfood 절차 정립 시 새 ADR로 supersede 가능.
