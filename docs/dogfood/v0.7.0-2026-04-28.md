## Mock dogfood 결과

### parseCliTargets (24 case PASS)
- default: `undefined`/`""`/`[]` → `["claude"]`
- single: `claude`/`codex`/`opencode` → 3 case
- repeatable: `[codex, claude]` → sorted `["claude", "codex"]`. 모든 7 combination
- alias deprecation: `both` → `["claude", "codex"]` + warning, `all` → `["claude", "codex", "opencode"]` + warning
- alias + base mixed: `[both, opencode]` → all 3 + 1 warning
- duplicates: `[claude, claude]` → unique `["claude"]`
- invalid: `rust`/`claud`/`[claude, rust]` → ok=false + error

### 매트릭스 11×7=77 + invariant 8 PASS

11 Track × 7 combination:
- `[claude]`, `[codex]`, `[opencode]`, `[claude, codex]`, `[claude, opencode]`, `[codex, opencode]`, `[claude, codex, opencode]`

각 시나리오 검증:
- baseline `.claude/CLAUDE.md` + `settings.json` + `.mcp.json` (항상)
- `installedTracks` 정합성
- AGENTS.md 존재 = (codex || opencode 포함 여부)
- `.codex/config.toml` 존재 = (codex 포함 여부)
- `opencode.json` 존재 = (opencode 포함 여부)

신규 조합 검증:
- `[claude, opencode]` (Codex 제외): `.codex/config.toml` **없음**, `opencode.json` **있음**, AGENTS.md **있음**, `.claude/CLAUDE.md` **있음**
- `[codex, opencode]` (Claude 제외): baseline은 항상 `.claude/CLAUDE.md` 생성 (의도) + Codex/OpenCode 둘 다 산출

### Codex prompts (5 + 4 case PASS)

**`renderCodexPrompt`**:
- frontmatter description 보존
- `/uzys:<phase>` → `/uzys-<phase>` slash rename
- frontmatter 부재 시 default description fallback
- double-quote escape

**`runCodexOptIn` withCodexPrompts**:
- true + harnessRoot 제공 → 6 prompts 생성 (~/.codex/prompts/uzys-*.md)
- false → 0 prompts (opt-out 강제)
- harnessRoot 부재 → projectDir/.claude/commands/uzys fallback
- idempotent: 2회 호출 시 동일 결과
